# 09 · Governance and Compliance

## 1. Concepto general

**Governance** y **Compliance** definen cómo una organización establece, controla y garantiza el cumplimiento de políticas, leyes y estándares de seguridad.

### Governance (Gobernanza)

Dirección estratégica del programa de seguridad.
Define **cómo se protegen los activos** y **quién es responsable**.

Ejemplos:

* Políticas internas
* Roles y responsabilidades
* Gestión de riesgos

### Compliance (Cumplimiento)

Grado en que la organización cumple leyes, regulaciones o estándares.

Ejemplos:

* GDPR
* HIPAA
* PCI-DSS

---

## 2. Elementos de gobernanza de seguridad

| Elemento                        | Propósito                  | Ejemplo                                |
| ------------------------------- | -------------------------- | -------------------------------------- |
| **Policies (Políticas)**        | Directrices de alto nivel  | “Todo dispositivo debe usar AES-256”   |
| **Standards (Estándares)**      | Cómo cumplir la política   | “Usar BitLocker para cifrado”          |
| **Procedures (Procedimientos)** | Instrucciones paso a paso  | “Cómo activar BitLocker en Windows 10” |
| **Guidelines (Directrices)**    | Recomendaciones opcionales | “Cambiar contraseñas cada 90 días”     |

---

## 3. Marcos y estándares de referencia (Frameworks)

Frameworks utilizados para establecer, medir y mejorar la seguridad:

| Framework             | Enfoque                     | Descripción breve                                                           |
| --------------------- | --------------------------- | --------------------------------------------------------------------------- |
| **NIST CSF**          | Ciberseguridad general      | 5 funciones: Identify, Protect, Detect, Respond, Recover                    |
| **ISO 27001 / 27002** | Gestión de seguridad (ISMS) | Amplio uso internacional                                                    |
| **COBIT**             | Gobierno de TI              | Integra objetivos del negocio y control de TI                               |
| **ITIL**              | Operaciones TI              | Mejores prácticas de gestión de servicios                                   |
| **SOC 2 (AICPA)**     | Seguridad en proveedores    | Evalúa seguridad, disponibilidad, integridad, confidencialidad y privacidad |

---

## 4. Cumplimiento legal y normativo

Ejemplos comunes:

* **GDPR** (Europa): protección de datos personales
* **HIPAA** (EE. UU.): datos médicos
* **PCI-DSS**: transacciones con tarjetas
* **SOX**: control financiero corporativo
* **FISMA / FedRAMP**: sistemas federales y nube

**Principio clave:** la empresa sigue siendo responsable incluso si delega en terceros (**accountability**).

---

## 5. Roles y responsabilidades en gobernanza

| Rol                             | Responsabilidad                              |
| ------------------------------- | -------------------------------------------- |
| **CISO**                        | Lidera estrategia de seguridad               |
| **Data Owner**                  | Define sensibilidad y controles necesarios   |
| **Data Custodian**              | Administra y mantiene seguridad de los datos |
| **Privacy Officer / DPO**       | Cumplimiento legal y privacidad              |
| **Security Analyst / Engineer** | Implementa y monitorea controles             |
| **Auditor Interno/Externo**     | Verifica cumplimiento y eficacia             |

---

## 6. Gestión de riesgos (Risk Management)

Proceso fundamental dentro de la gobernanza.

### Etapas

* Identificación
* Evaluación
* Mitigación
* Monitoreo

### Estrategias

* **Evitar** → eliminar el riesgo
* **Mitigar** → reducirlo mediante controles
* **Transferir** → seguros o outsourcing
* **Aceptar** → asumirlo documentadamente

---

## 7. Auditorías, controles y documentación

* **Auditoría:** evaluación independiente del cumplimiento
* **Controles internos:** técnicos o administrativos
* **Documentación:** evidencia de políticas, registros, reportes, capacitación

Ejemplos:

* Informes de escaneos de vulnerabilidades
* Registros de formación del personal

---

## 8. Mejores prácticas de gobernanza

* Definir roles claros y líneas de reporte
* Revisar políticas anualmente
* Mantener inventario de activos y clasificación de datos
* Alinear seguridad con los objetivos del negocio
* Promover cultura de cumplimiento mediante capacitación
